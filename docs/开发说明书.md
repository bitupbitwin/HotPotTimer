# 火锅捞捞 · 设计与开发说明书

> 文档版本：v1.0　|　适用应用版本：1.0.x　|　最后更新：2026-06-28
> 包名：`com.laolao.hotpot_timer`　|　平台：Android（竖屏手机）

---

## 一、项目概述

### 1.1 一句话定位
**火锅捞捞** 是一款离线、无广告、无登录的火锅涮菜计时助手。用户在涮火锅时，点一下食材圆圈即可开始倒计时，到达最佳火候自动用声音 + 震动提醒，避免涮老或没熟。

### 1.2 解决的痛点
- 不同食材最佳涮煮时间差别大（毛肚 15 秒 vs 土豆数分钟），凭感觉容易翻车。
- 多人火锅同时下多种菜，靠脑子记不过来。
- 想知道某食材配什么蘸料、有忌口时怎么避雷。

### 1.3 设计原则
| 原则 | 落地方式 |
|------|---------|
| 极简上手 | 点圆圈即开始，无需阅读说明 |
| 离线优先 | 所有数据本地存储，无服务器、无网络依赖 |
| 隐私安全 | 不收集任何个人信息，无第三方 SDK |
| 多感官提醒 | 声音 + 震动 + 颜色三重反馈，嘈杂环境也能感知 |
| 可扩展 | 食材、蘸料、图片均可自定义/替换 |

## 二、目标用户与场景

- **主要用户**：经常在家或外出吃火锅的年轻人、家庭。
- **核心场景**：手机放在桌边，涮菜时随手点按，余光看颜色、听提示音。
- **设备要求**：Android 手机，竖屏，强制锁定竖屏（见 `main.dart`）。

## 三、功能需求

### 3.1 功能总览（三个 Tab）

底部导航三个页面（默认进入"涮锅"）：

| Tab | 名称 | 职责 |
|-----|------|------|
| 1 | 🥣 蘸料 | 忌口设置、预设/自定义蘸料方案 |
| 2 | 🍲 涮锅 | **核心**：选菜 + 计时（默认页） |
| 3 | 👤 我的 | 偏好、使用说明、意见反馈、隐私政策、版本 |

### 3.2 涮锅页（核心）功能点

1. **左侧分类栏**：全部 / 已点 / 推荐 / 牛肉 / 羊肉 / 内脏 / 丸滑豆制 / 海鲜 / 青菜 / 菌菇 / 主食 / 自定义。
2. **食材网格**：两列卡片，仅显示头像 + 菜名（紧凑布局，4 字以上不截断）。点击加入"已点"。
3. **已点计时区**：以圆圈形式展示每个已点食材，点圈开始/重置计时；按超时程度自动排序（超时＞将熟＞计时中＞未开始）。
4. **照片识别（占位实现）**：从相册选图 + 文字补充，匹配已收录食材加入已点。当前为关键词匹配占位层，预留接入真实视觉 API 的接口。
5. **自定义食材**：手动新增名字 + 倒计时秒数，本地持久化。
6. **数量叠加**：同一食材多次加入显示 `xN`。

### 3.3 计时状态机（关键逻辑）

定义见 `models/hotpot_item.dart` 的 `HotpotState`，判定见 `selected_hotpot_item.dart`：

| 状态 | 触发条件 | 外圈表现 | 反馈 |
|------|---------|---------|------|
| `idle` 未下锅 | 未点击开始 | 黑色 | — |
| `counting` 煮制中 | `0 ≤ 用时 < 推荐时间` | 黄色闪烁 | 点击开始时：上升提示音 + 轻震 |
| `ready` 完美 | `推荐 ≤ 用时 < 推荐+60s` | 绿色常亮 | 叮咚音 + 中震（间歇） |
| `overcooked` 超时 | `用时 ≥ 推荐+60s` | 红色高频闪烁 | 急促三连音 + 连续震动 |

> "完美窗口"固定为推荐时间后的 60 秒。

### 3.4 蘸料页功能点
- 忌口 FilterChip：勾选忌口食材，全 App 同步。
- 5 个预设蘸料方案卡片（含忌口高亮提示）。
- 自定义蘸料：增/删，本地保存。

## 四、技术架构

### 4.1 技术栈
| 层 | 选型 |
|----|------|
| 框架 | Flutter（Dart SDK `^3.10.7`），Material 3，暗色主题 |
| 状态管理 | `StatefulWidget` + `setState`（轻量，无第三方状态库） |
| 本地存储 | `shared_preferences`（JSON 序列化） |
| 多媒体 | `audioplayers`（提示音）、`vibration`（震动） |
| 取图 | `image_picker` |
| 构建 | Gradle 8.10.2 + AGP 8.7.3 + Kotlin 2.1.0 |

### 4.2 架构分层

```
lib/
├── main.dart                  # 入口：锁竖屏、初始化反馈服务、MaterialApp
├── models/                    # 数据模型（纯 Dart，可序列化）
│   ├── hotpot_item.dart       #   食材静态配置 + 状态枚举
│   ├── selected_hotpot_item.dart # 已点食材（含 startedAt 运行时状态、状态计算）
│   └── sauce_recipe.dart      #   蘸料方案
├── data/                      # 内置静态数据
│   ├── default_items.dart     #   20 个默认食材 + 分类常量
│   └── default_sauces.dart    #   5 个默认蘸料
├── services/                  # 无 UI 的逻辑/IO 层
│   ├── feedback_service.dart  #   声音 + 震动统一封装
│   ├── food_recognition_service.dart # 图片识别（占位，预留 API 接口）
│   ├── custom_item_store.dart #   自定义食材持久化
│   └── selected_item_store.dart # 已点列表持久化
├── widgets/
│   └── hotpot_item_widget.dart # 计时圆圈组件（状态驱动动画 + 触发反馈）
└── screens/
    ├── main_scaffold.dart     # 底部导航容器，托管跨页共享状态
    ├── home_screen.dart       # 涮锅页（核心）
    ├── seasoning_screen.dart  # 蘸料页
    ├── profile_screen.dart    # 我的页
    └── privacy_policy_screen.dart # 隐私政策
```

**依赖方向**：`screens → widgets → services → models → data`，单向，无循环依赖。

### 4.3 状态共享方式
`MainScaffold` 用 `IndexedStack` 承载三页以保留各页状态，并集中持有跨页数据（忌口集合 `tabooItems`、自定义蘸料 `customSauces`、当前 Tab），通过构造参数下发给子页。

## 五、数据模型

### 5.1 HotpotItem（食材）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 唯一标识（如 `1`、`custom_3`） |
| name | String | 食材名 |
| category | String | 分类 |
| targetSeconds | int | 推荐煮制秒数 |
| aliases | List\<String> | 别名（用于识别匹配） |
| imagePath | String? | 本地图片路径，空则用 emoji |
| emoji | String | 兜底显示 |

支持 `fromJson / toJson / copyWith`。

### 5.2 SelectedHotpotItem（已点食材）
- 组合 `HotpotItem` + `quantity` + `startedAt`。
- 由 `startedAt` 与当前时间动态计算 `state / elapsedSeconds / remainingSeconds / overtimeSeconds`，**不存储**派生状态，保证一致性。

### 5.3 SauceRecipe（蘸料）
`id / name / tag / description / ingredients[] / isCustom`。

### 资源替换约定
- 食材图：放入 `assets/images/`，命名 `<id>_<拼音>.png`（如 `1_maodu.png`），PNG，建议正方形 ≥256×256。
- 提示音：`assets/sounds/{start,ready,overcooked}.wav`，可直接替换同名文件。

## 七、开发环境与构建

### 7.1 环境
- Flutter SDK（Dart `^3.10.7`）、Android Studio / JDK 17。
- Android 构建链：Gradle 8.10.2、AGP 8.7.3、Kotlin 2.1.0（已在 `android/settings.gradle.kts`、`gradle-wrapper.properties` 固定）。

### 7.2 常用命令
```bash
flutter pub get                 # 安装依赖
flutter run                     # 真机/模拟器调试
flutter analyze                 # 静态检查
flutter build apk --release     # 出 release APK
```
产物：`build/app/outputs/flutter-apk/hotpot-<版本>-release.apk`（已配置自定义命名）。

### 7.3 签名
- 模板：`android/key.properties.template` → 复制为 `key.properties`（已 gitignore）填真实信息。
- 无 `key.properties` 时回退 debug 签名，**仅供自测，不可上架**。

### 7.4 版本号自动化
`.claude/bump-version.sh` + `PreToolUse` 钩子：每次提交自动把 `pubspec.yaml` 的 patch 与 build 号 +1（如 `1.0.0+1 → 1.0.1+2`），保证 Android `versionCode` 递增。

### 7.5 release 注意
`isMinifyEnabled = false`、`isShrinkResources = false`（开启会被 R8 裁掉 Flutter 引擎导致启动崩溃，已固化在 `app/build.gradle.kts`）。

## 十一、风险与注意

1. **图片识别为占位实现**：当前仅按文件名/输入关键词匹配，真实识别需接入视觉 API，注意届时更新隐私政策（涉及上传图片）。
2. **构建链版本敏感**：AGP/Gradle 过新会触发 `compileSdk` 强制声明导致插件报错，升级前先验证。
3. **后台计时**：当前依赖前台 1 秒 ticker，App 切后台被系统冻结时计时不走真实后台通知，重回前台按 `startedAt` 重新计算（逻辑正确，但无后台推送提醒）。如需后台提醒，需引入本地通知。
4. **数据仅本地**：卸载即清空，无备份。
